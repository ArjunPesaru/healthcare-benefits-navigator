# MA Health Benefits Navigator
![Demo](assets/demo.gif)
A Retrieval-Augmented Generation (RAG) system for exploring Massachusetts health insurance plans (plan year 2025). Ask plain-English questions about coverage, copays, deductibles, and plan comparisons — and get answers grounded in real MA Health Connector data.

## Features

- Natural-language search over 30 MA health plans (HMO, PPO — Bronze → Platinum)
- FAISS vector index for semantic retrieval using `all-MiniLM-L6-v2` embeddings
- XGBoost re-ranker with 11 query-chunk relevance features (NDCG objective)
- Mistral Small LLM for final structured answer generation (JSON output)
- ConnectorCare subsidy eligibility for households up to 500% FPL
- Age-adjusted premium estimates using CMS 2:1 statutory cap curve
- Budget filter: exclude plans exceeding a user-set monthly premium limit
- Auto-detects HSA intent in queries for better Bronze plan retrieval
- Interactive US insurance marketplace dashboard (50 states + DC)
- User feedback logging to CSV for continuous improvement
- Data quality validation checks on every pipeline rebuild

## Setup

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Set your Mistral API key

Create a `.env` file in the project root:

```
MISTRAL_API_KEY=your_key_here
```

### 3. Build the index and train the re-ranker (run once)

```bash
python setup.py
```

This will:
- Generate plan CSVs from `config.py`
- Run data quality validation checks
- Embed all chunks using `all-MiniLM-L6-v2` and build the FAISS index
- Train the XGBoost re-ranker on 15 labelled training queries

### 4. Launch the app

```bash
streamlit run app.py
```

The app will be available at **http://localhost:8501**

## Evaluation

### Retrieval pipeline evaluation (NDCG@5, Hit@1, MRR)

Evaluates FAISS semantic search + XGBoost re-ranking on 10 held-out queries
that are distinct from the training set. No Mistral API call is made.

```bash
python evaluate.py
```

**Metrics reported:**
| Metric | Description |
|--------|-------------|
| NDCG@5 | Normalised Discounted Cumulative Gain at rank 5 |
| Hit@1  | Whether the top-ranked result is relevant |
| MRR    | Mean Reciprocal Rank across all queries |

### LLM output quality evaluation (BLEU + ROUGE)

Evaluates Mistral's generated answers against 8 human-authored reference answers
grounded in verified MA Health Connector plan data.

```bash
python evaluate_llm.py
```

**Metrics reported:**
| Metric  | Description |
|---------|-------------|
| BLEU-1  | Unigram precision overlap with reference answers |
| BLEU-2  | Bigram precision overlap with reference answers |
| ROUGE-1 | Unigram F1 recall/precision |
| ROUGE-2 | Bigram F1 recall/precision |
| ROUGE-L | Longest common subsequence F1 |

> **Note:** ROUGE-L ≥ 0.30 indicates strong factual alignment for open-domain conversational QA.

## Architecture

```
User Query
    │
    ▼
encode_query()          ← Prepend age/tier/carrier/budget context
    │
    ▼
FAISS semantic search   ← all-MiniLM-L6-v2 embeddings, cosine similarity (TOP_K=15)
    │
    ▼
filter_by_cost()        ← Remove plans exceeding monthly budget at user's age
    │
    ▼
XGBoost re-ranker       ← 11 features: FAISS score, keyword overlap, tier/carrier/type
    │                      match, ConnectorCare flag, premium query, age, feedback score
    ▼
Mistral Small LLM       ← Structured JSON: narrative answer + ranked plan table
    │                      temperature=0.1 for deterministic responses
    ▼
Streamlit UI            ← Chat interface + plan comparison table + ranking justifications
```

## Model Justification

| Component | Choice | Justification |
|-----------|--------|---------------|
| Embeddings | `all-MiniLM-L6-v2` | Lightweight (80 MB), strong semantic coverage of health domain vocabulary, fast inference |
| Re-ranker | XGBoost (rank:pairwise) | Interpretable, fast, works well on small labelled datasets; NDCG eval metric aligns with ranking objective |
| LLM | Mistral Small | Cost-effective for structured JSON output; temperature=0.1 ensures reproducible plan recommendations |

## Data Sources

- **CMS Benefits & Cost Sharing PUF 2025** — benefit categories and cost-sharing structure
- **MA Health Connector 2025** — ConnectorCare plan parameters (PDF Table 4)
- **MA DOI 2025** — licensed carrier list
- **CMS MA state-specific age curve** — age multipliers with 2:1 statutory cap (ACA §2701)
